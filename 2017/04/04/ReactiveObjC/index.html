ReactiveObjC

发表于 2017-04-04

注意：这是 Objective-C ReactiveCocoa 的传统介绍，现在称为 ReactiveObjC。 对于使用 Swift 的更新版本，请参阅ReactiveCocoa或ReactiveSwift

ReactiveObjC（正式的名称 ReactiveCocoa 或 RAC）是由函数响应式编程启发的Objective-C框架。 它提供用于编写和转换流媒体值的API。

如果您已经熟悉了函数响应式编程，或者了解了ReactiveObjC的基本前提，请查看此文件夹中的其他文档以了解框架概述，以及有关在实践中如何运行的更深入的信息。

    ###介绍
    ReactiveObjC的灵感来自函数响应式编。 RAC提供的信号（由RACSignal表示）可以捕获当前和未来的值，而不是使用被替换和修改的可变变量。

通过链接，组合和响应信号，软件可以声明性地编写，而不需要持续观察和更新值的代码。

例如，文本字段可以绑定到最新的时间，即使它改变了，而不是使用附加的代码来监视时钟并且每秒更新文本字段。它非常像KVO，但是使用块而不是重写方法-observeValueForKeyPath：ofObject：change：context：。

信号也可以处理异步操作，非常像期货和承诺。这极大地简化了异步软件，包括网络代码。

RAC的主要优点之一是它提供了一种单一的统一方法来处理异步行为，包括委托方法，回调块，目标动作机制，通知和KVO。

系统要求

ReactiveObjC supports OS X 10.8+ and iOS 8.0+.

Importing ReactiveObjC

要将RAC添加到应用程序中：

  1. 添加 ReactiveObjC 仓库为你应用仓库的子模块
  2. 运行 git 子模块 update–init –recursive 在 ReactiveObjC 文件夹内
  3. 将 ReactiveObjC.xcodeproj 拖放到应用程序的Xcode项目或工作区中。
  4. 在 你应用 target 的 “Build Phases” 选项卡 , 添加 RAC 到 add “Link Binary With Libraries” phase.
  5. Add ReactiveObjC.framework. RAC must also be added to any “Copy Frameworks” build phase. If you don’t already have one, simply add a “Copy Files” build phase and target the “Frameworks” destination.
  6. Add “$(BUILD_ROOT)/../IntermediateBuildFilesPath/UninstalledProducts/include” $(inherited) to the “Header Search Paths” build setting (this is only necessary for archive builds, but it has no negative effect otherwise).
  7. For iOS targets, add -ObjC to the “Other Linker Flags” build setting.
  8. If you added RAC to a project (not a workspace), you will also need to add the appropriate RAC target to the “Target Dependencies” of your application.
