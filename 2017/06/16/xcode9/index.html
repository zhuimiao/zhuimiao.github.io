xcode9 发行说明

运行环境

Xcode 9 beta 需要运行在安装了 macOS 10.12.4 或更新系统的mac上.
Xcode 9 beta 包含 iOS 11, watchOS 4, macOS 10.13, and tvOS 11 的 SDks.

安装

Xcode 9 beta 能和之前版本的 Xcode 共存

预发布版本的Xcode可以从 developer.apple.com 获取，打包成了一个zip压缩文件。在测试阶段可以通过下载zip文件，双击解压，然后拖动到Applications 文件夹来安装Xcode。

直到正式版发布，Xcode才可以通过 App Store 安装。

访问附加开发工具

启动附加开发工具，比如：Instruments（性能调优使用）或 FileMerge （比较两份代码不同时使用）。
运行 Xcode-beta -> 选择 Xcode > Open Developer Tool。
您也可以把这些附加开发工具保存到 dock来直接使用，Xcode无需启动。

技术支持和学习资源

苹果提供了以下的网页资源支持你用Xcode进行开发。

  • 苹果开发者论坛. 参加开发苹果平台和使用开发工具的讨论
  • Bug Reporter. 向苹果报告问题，增强要求和反馈。提供详细的信息，包括系统和开发人员工具版本信息，以及任何相关信息
    崩溃日志或控制台消息。
  • 苹果开发者网站. 获取最新的开发信息和关于Xcode的技术文档。
  • Xcode主页。获得关于Xcode的最新发布的高级信息。下载
    电流和β的Xcode的版本
  • 使用Xcode的帮助，使用内置的帮助文档，点击 help > Xcode help

过期和移除提示

  • iOS 11 不在支持 armv7 和 armv7s (32-bit)。如果一个 iOS target 部署为 11，将不会被构建成32位。iOS target 部署比11低，仍会默认构建为32位。（说人话：iPhone5不支持了）
  • Xcode Server no longer includes the capacity to host Git repositories. To continue using repositories that were set up in macOS Server, configure a new remote and push your code to that remote. After upgrading to Xcode 9, any existing repositories will be archived in /Library/Developer/XcodeServer/HostedRepositories. (31243129)

新功能

通用

  • Xcode 9.0 支持同时运行多个Xcode app 和 任意相关工具模拟器（7，7plus，se，6同时运行四个模拟器）。Xcode 9.0 不能与前面版本同时运行。

  • -runFirstLaunch, a new flag in xcodebuild interactively checks the status of the license agreement and install any packages required for launching Xcode.

  • A file with a base name of FooTest or FooTests is now considered a counterpart of a file with a base name of Foo for navigation and the Assistant Editor.

源代码编辑器

  • 选择一段文本，在开头输入分隔符将在最后添加一个对应的关闭分隔符。 自动匹配的分隔符包括“”，（），[]，{}

  • 改变字体大小：command + 增大 ， command - 减小。

  • 在“首选项”的“字体和颜色”窗格中添加一个选择器，以修改源编辑器行间距。

  • 现在可以从3个游标中选择，使用源代码编辑器。
